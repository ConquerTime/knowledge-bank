## 🎯 使用指南

### 学习顺序建议

**技术基础薄弱者**：01 → 02 → 08 → 03 → 04 → 05 → 07 → 06 → 09 → 10

**有AI经验者**：03 → 04 → 05 → 07 → 06 → 01 → 02 → 08 → 09 → 10

**侧重行为面试**：09 → 10 → 06 → 其他

### 学习方法

1. **先自己回答**：看到问题后，先自己思考和组织答案
2. **对比参考答案**：看看示例答案的思路和结构
3. **提取关键点**：总结每个问题的考察点和回答要点
4. **模拟练习**：找人模拟面试，或者对着镜子/录音练习
5. **持续迭代**：根据反馈不断改进答案

## 📚 各面试类型详解

### 技术筛选面试（01-02）

**目的**：验证基础技术能力

**准备重点**：
- 熟练掌握主要技术栈的核心概念
- 能够写出正确且规范的代码
- 了解常见的性能优化方法

**常见失分点**：
- 基础概念模糊
- 只会用不理解原理
- 代码习惯不好

### AI技术深度面试（03-05）

**目的**：验证AI领域的专业能力

**准备重点**：
- 理解LLM、RAG、Agent的核心原理
- 有实际的项目经验和踩坑教训
- 了解主流工具和框架的特点

**常见失分点**：
- 只停留在API调用层面
- 缺乏生产环境的实战经验
- 对安全和成本考虑不足

### 项目深挖面试（06）

**目的**：深入了解过往项目经验

**准备重点**：
- 用STAR结构组织项目介绍
- 准备2-3个代表性项目的详细讲解
- 能够解释技术决策的原因
- 准备好量化的成果数据

**常见失分点**：
- 讲不清楚自己的具体贡献
- 无法解释技术选型的依据
- 缺乏反思和改进意识

### 系统设计面试（07）

**目的**：考察系统架构能力

**准备重点**：
- 掌握系统设计的回答框架
- 熟悉常见的架构模式
- 能够做出合理的技术权衡
- 了解AI系统的特殊考量

**常见失分点**：
- 没有澄清需求就开始设计
- 设计过于理想化，不考虑约束
- 只关注技术，忽视业务

### 算法编程面试（08）

**目的**：验证编码能力和算法思维

**准备重点**：
- 常见数据结构和算法
- 代码质量和规范
- 复杂度分析
- AI场景的特殊算法

**常见失分点**：
- 紧张导致思路混乱
- 不与面试官沟通就开始写
- 代码写完不测试

### 行为面试（09）

**目的**：了解软技能和工作风格

**准备重点**：
- 准备各类行为问题的真实案例
- 用STAR结构组织回答
- 展示自我认知和反思能力

**常见失分点**：
- 案例不具体，泛泛而谈
- 只说做了什么，不说学到什么
- 回避缺点或失败经历

### 综合面试/终面（10）

**目的**：判断文化契合和长期发展潜力

**准备重点**：
- 对行业有自己的思考
- 清晰的职业规划
- 展示领导潜力
- 了解目标公司

**常见失分点**：
- 对公司了解不够
- 职业规划不清晰或不现实
- 过于被动，缺乏主动性

## 💡 面试技巧总结

### 通用技巧

1. **结构化回答**：使用STAR或其他框架组织回答
2. **时间控制**：注意面试官的提示，不要说太多
3. **互动沟通**：不确定的地方主动确认
4. **展示思考**：说出你的思考过程
5. **保持诚实**：不会的就说不会，但展示学习意愿

### 技术面试技巧

1. **先思考再编码**：理清思路后再动手
2. **边写边说**：让面试官了解你的思路
3. **测试验证**：写完代码主动测试
4. **考虑边界**：处理空值、边界情况
5. **优化讨论**：主动讨论优化空间

### 行为面试技巧

1. **准备案例库**：提前准备各类问题的案例
2. **量化结果**：用数据说明成果
3. **展示成长**：强调从经历中学到什么
4. **保持正面**：即使是负面经历也要正面表达
5. **真实自然**：不要过度包装，真诚最重要
